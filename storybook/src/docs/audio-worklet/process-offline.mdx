# processOffline

**What is this?**

`processOffline` renders an entire `AudioBuffer` through SoundTouch processing in a single awaited call. It uses `OfflineAudioContext` so no real-time audio device is needed — useful for exporting or pre-processing audio files.

**When would I use it?**

- Exporting a pitch-shifted or tempo-adjusted clip to a downloadable file
- Pre-processing audio before playback without a live audio graph
- Server-side or headless browser environments

## Import

```ts
import { processOffline } from '@soundtouchjs/audio-worklet';
```

## Signature

```ts
async function processOffline(options: ProcessOfflineOptions): Promise<AudioBuffer>
```

## Options

```ts
interface ProcessOfflineOptions {
  input: AudioBuffer;               // source audio to process
  processorUrl: string | URL;       // URL to soundtouch-processor.js
  pitch?: number;                   // default 1.0
  pitchSemitones?: number;          // default 0
  playbackRate?: number;            // default 1.0; also scales output length
  interpolationStrategy?: RateTransposerInterpolationStrategy;
  stretchParameters?: StretchParameters;
  sampleBufferType?: SampleBufferType;
}
```

| Option | Default | Description |
|--------|---------|-------------|
| `input` | — | Source `AudioBuffer` |
| `processorUrl` | — | Path to the bundled processor script |
| `pitch` | 1.0 | Pitch multiplier |
| `pitchSemitones` | 0 | Semitone offset (combined with `pitch`) |
| `playbackRate` | 1.0 | Speed multiplier; output length = `ceil(input.length / playbackRate)` |
| `interpolationStrategy` | `'lanczos'` | Rate transposer kernel |
| `stretchParameters` | — | WSOLA timing overrides (see [Stretch](../core/stretch)) |
| `sampleBufferType` | `'circular'` | Internal buffer strategy |

## Example

```ts
import { processOffline } from '@soundtouchjs/audio-worklet';

// Fetch and decode a source file
const response = await fetch('/audio.mp3');
const arrayBuffer = await response.arrayBuffer();
const audioCtx = new AudioContext();
const audioBuffer = await audioCtx.decodeAudioData(arrayBuffer);

// Process offline — no audio device needed
const processed = await processOffline({
  input: audioBuffer,
  processorUrl: '/soundtouch-processor.js',
  pitchSemitones: -3,
  playbackRate: 1.2,
});

// Play the result or export it
const source = audioCtx.createBufferSource();
source.buffer = processed;
source.connect(audioCtx.destination);
source.start();
```

## Notes

- The output `AudioBuffer` has the same number of channels and sample rate as the input.
- `playbackRate` affects both pitch compensation and output length estimation. For pitch-only changes, leave it at `1.0` and use `pitch` or `pitchSemitones`.
- `processorUrl` must point to a fully-bundled processor script. See [Getting Started](./getting-started) for bundler-specific instructions.
- The `OfflineAudioContext` does not support real-time automation; all `AudioParam` values are fixed at the time of rendering.
