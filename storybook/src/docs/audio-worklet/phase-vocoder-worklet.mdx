# PhaseVocoderNode

**What is this?**

`@soundtouchjs/phase-vocoder-worklet` provides an `AudioWorkletNode` backed by an FFT-based phase vocoder instead of the default WSOLA stretch algorithm. Use it as a drop-in replacement for `SoundTouchNode` when you need smoother time-stretching at extreme playback ratios.

**When would I use it?**

- Playback at extreme ratios (< 0.5× or > 2×) where WSOLA produces audible clicks or repeats.
- Smooth slow-motion or very fast playback where "phasiness" is acceptable.
- You need the full `SoundTouchNode` API (pitch, semitones, rate, interpolation strategy) with a different stretch core.

For moderate ratios (0.5–2×) `SoundTouchNode` typically sounds better.

## Setup

### Registering the processor

```ts
import { PhaseVocoderNode } from '@soundtouchjs/phase-vocoder-worklet';

const audioCtx = new AudioContext();
// Vite:
import processorUrl from '@soundtouchjs/phase-vocoder-worklet/processor?url';
await PhaseVocoderNode.register(audioCtx, processorUrl);
```

See [Getting Started](./getting-started) for Vite, webpack, and static setups.

## Basic usage

```ts
const node = new PhaseVocoderNode({
  context: audioCtx,
  fftSize: 2048,     // default
  overlapFactor: 4,  // default
});

node.pitch.value = 1.5;        // 50 % pitch up
node.pitchSemitones.value = 3; // shift by semitones
node.playbackRate.value = 0.25; // extreme slow-down — smooth with phase vocoder

node.connect(audioCtx.destination);
sourceNode.connect(node);
```

## Constructor options

```ts
new PhaseVocoderNode({
  context: audioCtx,
  fftSize?: 512 | 1024 | 2048 | 4096,  // default: 2048
  overlapFactor?: 2 | 4 | 8,           // default: 4
  outputChannelCount?: 1 | 2,          // default: 2
  sampleBufferType?: 'circular' | 'fifo',
  interpolationStrategy?: RateTransposerInterpolationStrategy,
})
```

## Offline processing

```ts
import { processOffline } from '@soundtouchjs/phase-vocoder-worklet';

const processed = await processOffline({
  input: audioBuffer,
  processorUrl,
  playbackRate: 0.5,
  pitchSemitones: 3,
  fftSize: 1024,
  overlapFactor: 8,
});
```

## Processor observability

The render-thread processor posts metrics every 100 render blocks:

```ts
node.addEventListener('metrics', (e) => {
  const { framesBuffered, underrunCount, blockCount, outputRms, outputPeak } =
    (e as CustomEvent<ProcessorMetrics>).detail;
  // e.g. display outputRms/outputPeak in your UI
});

// Or poll the getter:
const m = node.metrics; // ProcessorMetrics | null
```

`ProcessorMetrics`:

<table>
  <thead>
    <tr><th>Field</th><th>Description</th></tr>
  </thead>
  <tbody>
    <tr><td><code>framesBuffered</code></td><td>Output frames available at the last render block</td></tr>
    <tr><td><code>underrunCount</code></td><td>Cumulative blocks with fewer output frames than requested</td></tr>
    <tr><td><code>blockCount</code></td><td>Total render blocks processed since creation</td></tr>
    <tr><td><code>outputRms</code></td><td>Root mean square (RMS) of the last output block (both channels)</td></tr>
    <tr><td><code>outputPeak</code></td><td>Peak absolute value of the last output block (both channels)</td></tr>
    <tr><td><code>timestamp</code></td><td><code>performance.now()</code> on the main thread when metrics arrived</td></tr>
  </tbody>
</table>

## Trade-offs vs `SoundTouchNode`

<table>
  <thead>
    <tr><th></th><th><code>SoundTouchNode</code> (WSOLA)</th><th><code>PhaseVocoderNode</code></th></tr>
  </thead>
  <tbody>
    <tr><td>Quality at extreme ratios</td><td>Artifacts above 2×</td><td>Smooth at all ratios</td></tr>
    <tr><td>Transient preservation</td><td>Better (time-domain)</td><td>Worse (frequency smearing)</td></tr>
    <tr><td>Computation</td><td>Lower</td><td>Higher (FFT per hop)</td></tr>
    <tr><td>Startup latency</td><td>Lower</td><td><code>fftSize</code> samples</td></tr>
    <tr><td>Artifacts</td><td>Clicks / repeats</td><td>"Phasiness" / smearing</td></tr>
  </tbody>
</table>
