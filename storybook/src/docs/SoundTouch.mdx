# SoundTouch

**What is this?**

This is the core engine that lets you shift the pitch or key of audio in one place. It combines the `RateTransposer` and `Stretch` stages so you don’t have to wire them up yourself.

**When would I use it?**

Use this directly only when building a custom audio pipeline. For browser playback, use [`SoundTouchNode`](../audio-worklet/sound-touch-node) from `@soundtouchjs/audio-worklet` instead.

`SoundTouch` is the main processing engine that chains `RateTransposer` and `Stretch` for real-time pitch manipulation.

## Import

```ts
import { SoundTouch } from '@soundtouchjs/core';
```

## Constructor

```ts
new SoundTouch({
  sampleRate?: number,
  sampleBufferType?: 'fifo' | 'circular',
  sampleBufferFactory?: SampleBufferFactory,
  interpolationStrategy?: RateTransposerInterpolationStrategy,
})
```

## Public API

- virtualPitch (field) — current pitch multiplier; updated by the pitch setters
- pitch (setter)
- pitchOctaves (setter)
- pitchSemitones (setter)
- inputBuffer (getter)
- outputBuffer (getter)
- interpolationStrategy (getter)
- interpolationStrategyParams (getter)
- setInterpolationStrategy(strategy)
- setInterpolationStrategyParams(params)
- setStretchParameters(params: StretchParameters) — update WSOLA timing on the stretch stage
- clear()
- clone()
- calculateEffectiveRateAndTempo()
- process()

## setStretchParameters

```ts
import { SoundTouch } from '@soundtouchjs/core';
import type { StretchParameters } from '@soundtouchjs/core';

const st = new SoundTouch({});
st.setStretchParameters({ overlapMs: 12, quickSeek: false });
```

Delegates to `Stretch.setStretchParameters`. See [Stretch](./stretch) for the full parameter reference.

> **Note:** `tempo` and `rate` are internal pipeline values derived from `virtualPitch` — they are not part of the public API. For browser playback, use [`SoundTouchNode`](../audio-worklet/sound-touch-node), which manages the pipeline internally.
