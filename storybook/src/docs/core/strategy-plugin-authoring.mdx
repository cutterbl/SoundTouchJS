# Strategy Plugin Authoring

Want to make your own interpolation strategy? You don’t need to be a DSP (digital signal processing) expert—just a bit of TypeScript and some curiosity!

This guide walks you through making a plugin that tells SoundTouchJS how to "fill in the blanks" when changing audio speed or pitch. You can experiment with your own math, or adapt ideas from other audio libraries.

**Why would I do this?**

- You want a different sound (maybe sharper, softer, or just "weird")
- You want to experiment with new algorithms
- You want to optimize for a special use case (like speech, or very low CPU)

You don’t need to know everything about audio—just follow the steps and listen to the results!

## 1) Define Strategy Types

```ts
export interface MyStrategyParams {
  sharpness: number;
}
```

## 2) Implement a Kernel

```ts
export const myKernel = (
  src: Float32Array,
  srcOffset: number,
  numFrames: number,
  position: number,
  channel: 0 | 1,
  state: unknown,
): number => {
  // interpolate one output sample
  return src[
    srcOffset +
      2 * Math.min(numFrames - 1, Math.max(0, Math.floor(position))) +
      channel
  ];
};

myKernel.createState = () => ({
  prevSampleL: 0,
  prevSampleR: 0,
  params: { sharpness: 1 },
});
```

## 3) Add Defaults, Validation, and Apply Hook

```ts
const defaultParams = { sharpness: 1 };

function normalizeParams(
  params: Partial<Record<string, number>> | undefined,
  defaults: Record<string, number>,
) {
  const merged = { ...defaults, ...(params ?? {}) };
  return {
    sharpness: Math.max(0, Math.min(4, merged.sharpness ?? 1)),
  };
}

function applyParams(state: unknown, params: Record<string, number>) {
  if (typeof state !== 'object' || state === null) {
    return;
  }
  (state as { params: { sharpness: number } }).params = {
    sharpness: params.sharpness ?? 1,
  };
}
```

## 4) Register the Strategy

```ts
export const myStrategy = {
  id: 'my-plugin/strategy',
  baseStrategy: 'linear' as const,
  kernel: myKernel,
  defaultParams,
  normalizeParams,
  applyParams,
};

export function registerMyStrategy(registry: {
  registerInterpolationStrategy: (registration: typeof myStrategy) => void;
}) {
  registry.registerInterpolationStrategy(myStrategy);
}
```

## 5) Consume in Core

```ts
import { registerInterpolationStrategy } from '@soundtouchjs/core';
import { registerMyStrategy } from '@your-scope/interpolation-strategy-my-plugin';

registerMyStrategy({ registerInterpolationStrategy });
```

## 6) Consume in AudioWorklet

Create a worklet installer module that registers your strategy, then load it before creating `SoundTouchNode`:

```ts
await SoundTouchNode.registerStrategyModule(audioCtx, strategyModuleUrl);
```

## 7) Support Runtime Updates

Expose params that can be patched through:

- `SoundTouch#setInterpolationStrategyParams(...)`
- `SoundTouchNode#setInterpolationStrategyParams(...)`

## 8) Test Checklist

- registration and lookup
- defaults/normalization
- runtime strategy switch
- runtime params update
- finite output under repeated 128-frame blocks
