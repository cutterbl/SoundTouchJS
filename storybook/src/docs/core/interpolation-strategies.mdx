# Interpolation Strategies (For Everyone)

When you change speed or pitch, SoundTouchJS has to estimate in-between samples. That estimation is the interpolation strategy. Different strategies trade CPU cost, latency, and sound quality.

## Available strategies

- lanczos8 (default): high-quality windowed-sinc baseline for most music and voice content.
- linear: lowest complexity and typically lowest latency.
- hann8 (plugin): good general-purpose quality with moderate roll-off.
- blackman8 (plugin): better stopband rejection, with a slightly wider transition band.
- kaiser8 (plugin): tunable strategy with flexible quality/performance tradeoffs.

## Built-in vs plugin strategies

- Built-in: lanczos8 and linear are always available.
- Plugin: hann8, blackman8, and kaiser8 require registration before use.

## Core usage

```ts
import { SoundTouch, registerInterpolationStrategy } from '@soundtouchjs/core';
import { registerHannStrategy } from '@soundtouchjs/interpolation-strategy-hann';
import { registerBlackmanStrategy } from '@soundtouchjs/interpolation-strategy-blackman';
import { registerKaiserStrategy } from '@soundtouchjs/interpolation-strategy-kaiser';

registerHannStrategy({ registerInterpolationStrategy });
registerBlackmanStrategy({ registerInterpolationStrategy });
registerKaiserStrategy({ registerInterpolationStrategy });

const soundTouch = new SoundTouch({
  interpolationStrategy: {
    id: 'kaiser8',
    params: { radius: 6, beta: 10 },
  },
});

soundTouch.setInterpolationStrategy({ id: 'blackman8', params: { radius: 5 } });
soundTouch.setInterpolationStrategyParams({ radius: 4 });
```

## AudioWorklet usage

```ts
import { SoundTouchNode } from '@soundtouchjs/audio-worklet';
import strategyInstallerModuleUrl from '../worklet/interpolation-strategies.installers.ts?url';

const context = new AudioContext();
await SoundTouchNode.register(context, processorModuleUrl);
await SoundTouchNode.registerStrategyModule(context, strategyInstallerModuleUrl);

const node = new SoundTouchNode({
  context,
  interpolationStrategy: { id: 'hann8', params: { radius: 6 } },
});

node.setInterpolationStrategy({ id: 'kaiser8', params: { radius: 6, beta: 12 } });
node.setInterpolationStrategyParams({ beta: 10 });
```

## Strategy guidance

- Use lanczos8 for best default quality.
- Use linear when CPU and responsiveness are the priority.
- Use hann8 for a balanced quality/performance profile.
- Use blackman8 when stronger stopband rejection is more important than a tight transition.
- Use kaiser8 when you want to tune behavior explicitly.

## Parameter notes

- lanczos8: radius (2..8, default 4)
- linear: edgeHoldFrames (0..32, default 1)
- hann8: radius (2..8, default 4)
- blackman8: radius (2..8, default 4)
- kaiser8: radius (2..16, default 4), beta (0..20, default 8.6)
